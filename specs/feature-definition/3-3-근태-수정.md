### 3.3 근태 수정

- **대상**: HR 담당자, 관리자

- **수정 가능 필드**

- 출근 시각 (checkIn)
- 퇴근 시각 (checkOut)
- 근태 상태 (status)
- 비고/사유 (note)

- **운영 규칙**

- 과거 날짜 근태 보정 가능
- 수정 이력이 중요한 경우, 별도 로그 테이블 또는 감사 기능 필요
- 날짜와 직원은 수정 불가 (삭제 후 재등록)

- **구현 참조**

- `attendance-edit` (feature: edit flow)
- 기존 근태 데이터 로드 + 수정 폼
